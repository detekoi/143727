Testing windows with scrolled areas

There are cases where not all the content available on an application page is visible at the same time. For example the entire window may be scrollable, or a particular element in the window may be scrollable. In this case, some extra steps need to be taken to ensure that the image grabbed at a checkpoint includes all the information on the page and not only that part which is currently visible.

provides the following APIs to deal with this:

eyes:ForceFullPageScreenshot()  
eyes:setScrollToRegion()  
eyes:setStitchMode()  
eyesBase:setStitchOverlap()  

eyes:checkFrame()

Used for checking I-Frame elements
eyes:checkRegion()  
eyes:checkRegionInFrame()
