Batch summary view

This Page is Under-Construction

This view provides summary information at the batch level in chart form, and also allows some filtering of the results.

Note that unless stated otherwise, tests that are still running are not taken into account in the information displayed. In the cases were tests were aborted, the results reflect the results available for that test (before the abort occurred) , the fact that not all the steps in the test were processed is not taken into account in the figured displayed. The information displayed is as follows :

Baseline Status
Displays the number of tests in the Saved state. Graphically compares it relatively to the number of unsaved and unchanged testsTBD: Should the gray label be Unchanged and not Unsaved. Answer it refers to things that are not changed - but were not "Saved" - i.e accepted/rejected ??
Diff Status
Displays the number of Resolved tests (test that were accepted implicitly or explicitly, or rejected explicitly). Graphically compares the relative number of Accepted, Rejected and Unresolved steps.
Test
Displays the total number of tests. Graphically compares the number of tests that have completed, are still running and have aborted. TBD. Concurrency
Shows the number of concurrent sessions used when running this batch relative to the maximum number of concurrent sessions available to your team in your account. TBD: correct ?
Checkpoints
Shows the total number of checkpoints in the batch, with a graphic breakdown to checkpoints that matched the baseline, mismatched the baseline or are new (irrespective if and how they were resolved as pass or failed). TBD what about checkpoints that were missing (-),
Test Status
Shows the number of passed test, and compares them graphically to the number of failed and unresolved tests. TBD what about aborted ?
Browsers
Displays the number of tests (TBD test or steps?)  that Passed, Failed or are still Unresolved for every browser type that participated in the batch.
Operating systems
Displays the number of tests (TBD test or steps?)  that Passed, Failed and are still Unresolved for every operating system type that participated in the batch.
Viewports

Team member

Issues and Remarks
Shows the number of issues and remarks there are in the steps of the batch broken down to remarks and issues that are new to this batch run, and those that already exists from previous runs. Does not include remarks at the test level.
