Test manager Views

You can use the main panel view selection toolbar to choose what information is displayed in the main panel.

The information displayed in these views overlap to some extent. Each view is optimized to present information at a particular level (batch, test or step) or for a particular type of information, and provides functionality that is relevant at that level.

In the views that display the step thumbnail, all the step level functionality is available.

Batch summary

In this view you can see graphic summaries of key test characteristics. For example, the percentage of the steps that have been resolved, percentage of tests that have passed, what browsers and operating systems the tests ran on and more.

Batch summary view

 		No test or step operations or information

Test details

This view focuses on information at the test level. It provides a table of all the tests in the batch (one per row) along with some key information such as the test status, execution environment and the viewport sizes.

You can sort, group and filter the tests by various criteria. You can select a subset of the tests and do some operations on all of them in one step (e.g. accept/reject all steps, assign or delete selected tests).

There are also operations that you can do on only in this view, specially:

  • mark a test with a "star" so that you can search for them later.
  • Force the state of the test into a passed or failed state irrespective of the step status
  • Add test remarks
  • Assign a test to a team member

If you click a test then it expands to reveal a subset of it's step thumbnails. You can scroll to see the other steps.

Remarks and issues

In this view you can see all the remarks and issues that have been entered to steps in the current batch.

You can click on any issue or remark to enter the test editor with that issue or remark displayed.
