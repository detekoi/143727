Branching in

In many projects, the development team works on two or more versions of an application concurrently. For example, the team may be maintaining hot fixes of a production version while developing a Beta version of the next release of the software. In other workflows, features are developed on separate software version control branches and are then merged back into the main release branch after testing.

provides a mechanism called branches to support such workflows. Since each version of the software can have a different look and feel, provides you with a way to configure a test so that each version of the software is tested against a different baseline. You configure a test by calling the setBranch() method with the name of your branch. At run time, searchs for a matching baseline in the list of baselines associated with that branch. After the test completes, the test results and the baseline (if it is created or saved), will be associated with that branch. .
